# 费用说明（必读）

## 核心结论

**SmsVerifyKit 本身完全免费、开源（MIT）。**  
**短信费用由你在阿里云账户中直接向阿里云支付，与本项目作者无关。**

阿里云短信认证 **不是免费服务**。每成功发送或校验一次，都会计入你的阿里云账单。

---

## 阿里云怎么收费？

短信认证（API 版）有两种计费方式（以 [阿里云官方定价](https://help.aliyun.com/zh/pnvs/product-overview/product-pricing) 为准，价格可能调整）：

### 1. 按量付费（用多少付多少）

成功调用按 **阶梯单价 × 次数** 计费，次月出账扣费。

**短信认证 API 版参考单价（2026 年初官方文档）：**

| 月成功量 | 约单价 |
|----------|--------|
| ≤ 1,000 次 | ¥0.06 / 次 |
| 1,000 – 5,000 | ¥0.056 / 次 |
| 1万 – 5万 | ¥0.048 / 次 |
| 10万 – 50万 | ¥0.046 / 次 |
| 更高用量 | 继续阶梯下降 |

> **举例**：个人站点月发 500 条验证码 ≈ 500 × ¥0.06 ≈ **¥30/月**（量级仅供参考，以账单为准）。

### 2. 套餐包（买定量的条数）

在控制台购买「短信认证套餐包」，优先抵扣，超出部分按量计费。适合用量稳定的业务。

---

## 什么操作会扣费？

| 操作 | 是否计费 |
|------|----------|
| 成功发送验证码（`SendSmsVerifyCode` 返回 OK） | ✅ 计费 |
| 校验验证码（`CheckSmsVerifyCode`） | ✅ 通常计费（以阿里云规则为准） |
| 发送失败（号码非法、未开通等） | ❌ 一般不计 |
| 本地代码调试、未调 API | ❌ 不计 |

**测试环境用真实手机号测也会扣费。**

## 与本项目的关系

| 项目 | 谁负责 | 费用 |
|------|--------|------|
| SmsVerifyKit 源码、SDK、文档 | 开源社区 / aiyangdie | **¥0** |
| 阿里云短信认证能力 | 阿里云 | **按量或套餐包** |
| 你部署的服务器 | 你自己 | 服务器/VPS 费用 |

我们**不**：
- 代你开通阿里云
- 代你发短息
- 代收短信费
- 保证阿里云价格不变

我们**只**提供：
- 对接阿里云 API 的开源代码
- 多语言 SDK 与自托管网关
- 接入文档与最佳实践

---

## 个人开发者怎么控制成本？

1. **开发阶段**：少测真实号码；用限流防止恶意刷短信  
2. **上线前**：设置阿里云 [费用预警](https://usercenter2.aliyun.com/home/expense-control)  
3. **业务层**：图形验证码 / 登录失败锁定后再发短信  
4. **用量大时**：买套餐包比纯按量更划算  

---

## 账户欠费会怎样？

阿里云账户欠费时，短信认证服务会**暂停**，你的用户将无法收到验证码。请及时充值。

官方定价页：[号码认证产品定价](https://help.aliyun.com/zh/pnvs/product-overview/product-pricing)
