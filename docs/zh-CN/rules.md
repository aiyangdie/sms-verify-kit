# 产品规则与使用边界

下面用**大白话**说明 SmsVerifyKit 是什么、能做什么、不能做什么。接入前请完整阅读。

---

## 1. 这是什么？

SmsVerifyKit 是一套 **开源工具包**，帮你把「手机号 + 短信验证码」接到自己的网站、App、小程序里。

它**不是**：
- 短信平台（不能替你没阿里云账号就发短信）
- 托管 SaaS（默认需你自己部署或嵌入 SDK）
- 保证送达的运营商服务

它**是**：
- 阿里云「短信认证」的封装代码 + 文档 + 可选 HTTP 网关
- 给个人开发者、小团队快速做登录/注册/绑手机

---

## 2. 你需要具备什么？

| 条件 | 说明 |
|------|------|
| 阿里云账号 | 需实名认证 |
| 开通短信认证 | 在号码认证控制台一键开通 |
| 服务端环境 | PHP / Node / Python / Go 任一，或部署 Gateway |
| 合规意识 | 短信内容、用户隐私需符合当地法律 |

**不需要**：自己申请短信签名模板（系统模板已提供）。

---

## 3. 典型业务场景

| 场景 | SDK scene | 推荐流程 |
|------|-----------|----------|
| 手机号登录 | `login` | 发码 → 用户输入 → verify 通过 → 签发登录态 |
| 手机号注册 | `register` | 发码 → verify → 创建用户并绑定手机 |
| 绑定手机 | `bind_phone` | 已登录用户 → 发码 → verify → 写库 |
| 更换手机 | `change_phone` | 已登录 → 新号发码 → verify → 更新 |
| 找回密码 | `reset_password` | 发码 → verify → 允许重置密码 |

**重要**：SDK 只负责「发码 + 验码」。**登录态、用户表、Session/JWT 需你自己实现。**

---

## 4. 安全规则（必须遵守）

1. **AccessKey Secret 只能放服务端**，禁止写进前端、App 安装包、Git 仓库  
2. **发码接口必须限流**：同号 60s 间隔、同 IP 每小时上限（Gateway 需自行加中间件）  
3. **验码通过后再改账号数据**，不要「只填码不验就绑手机」  
4. **验证码不要写入日志**  
5. 生产环境 Gateway 务必配置 `api_key`  

---

## 5. 隐私与合规

- 你收集用户手机号，你是**数据控制者**，需有隐私政策并告知用途  
- 短信仅用于你声明的场景（登录、验证等），禁止用于营销垃圾短信  
- 中国境内需遵守《个人信息保护法》《网络安全法》及运营商规范  
- 本工具不提供用户数据存储；用户数据在你自己的数据库
